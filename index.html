Medicine-Service

Oct 12, 2020 in Project, Team-Project

Medicine-Service - 약 복용 알리미





프로젝트 주제






  • 아날로그 방식 개선 목적 : 식사시간을 기준으로 정하는 아날로그 방식에서 복용알림 서비스로의 개선 목적.

  • 복용효과 100% 받기 : 약 동력의 기반하여 최적의 재 복용 시간을 알려 줌.

  • 체내 약 효능 시기 인지하기 : 체내에 약의 효과가 작용하고 있는 것을 시각적 그래프로 알려 줌.




개발환경




Database




기능구현

  • 검색 기능

  • 복용 시작

  • 복용 기록

  • 복용 그래프

  • 약국 찾기 지도

  • 실시간 채팅 상담


    검색 기능


  • 공공데이터 에서 약 53000개의 약 정보를 파싱하여 DB에 저장함.

  • 공공데이터 에서 약 100개의 약 부작용 정보를 파싱하여 DB에 저장함.

  • Jquery의 AutoComplete 기능을 이용하여 검색 시 자동완성 기능을 추가함.




< 검색 시 >



< 검색 완료 시 >




복용 시작


  • 원하는 약을 선택한 후, 처방받은 복용주기를 입력함.

  • 입력한 약의 이름을 정보 테이블 , 부작용 테이블에 접근하여 부합하는 정보를 가져와서 리스트로 나타냄.





< 주기 입력 창 >




< 복용 리스트 출력 >





복용 기록


  • 복용이 완료된 약들은 복용 리스트에서 지워지고, History 페이지에 자동으로 이동하여 저장됨.



< History >




복용 그래프


  • 임의로 만든 약 동력 그래프를 시간에 따라 그리게 되고, 복용주기가 끝나면 그래프는 멈추도록 설계함.

  • 약의 효과가 발현하는 55% 지점부터 (임의로 지정) 약 효과가 발현하고 있다는 신호를 주기위해 gif 이미지를 띄워 표시함.

  • 입력한 재복용 주기만큼 시간이 지나면 사용자의 휴대전화로 재복용 알람 문자 메세지를 전송함.


















약국 찾기 지도


  • 공공데이터에서 약 22000개의 전국 약국정보를 파싱하여 DB에 저장함.

  • 해당 약국의 위치를 표시하기 위해서 Marker 기능을 사용하였고, 해당 마커의 정보를 표기하기 위해 InfoWindow 기능을 사용함.

  • Marker가 한 곳에 밀집되어 있을 때, 밀집된 Marker의 수를 표시하기 위해서 Clusterer 기능을 구현함.

  • 광역시 시청을 기준으로 버튼 클릭 시 , 해당 좌표로 바로 이동할 수 있도록 구현함.














실시간 채팅


  • Admin으로 접속 했을 때에는 관리자 채팅 표시가 나타나게 하고, 일반 유저일 경우 실시간 상담 로고만 나타나도록 설정함.

  • 관리자 채팅으로 접속 시, 일반 유저들이 작성 한 채팅창이 뜨게 작성하였고 실시간 채팅이 가능하도록 구현함.

  • 일반 유저가 실시간 상담을 종료하면 관리자 채팅에서의 채팅방도 사라지게 구현함.

BufferReader VS Scanner

Oct 04, 2020 in 알고리즘, 20.10.05

이번 포스팅에서는 BufferedReader와 Scanner의 차이에 대해서 공부한걸 정리해보겠습니다.

1. BufferedReader

BufferedReader는
InputStreamReader에 버퍼링 기능이 추가된것이다.

사용자가 요청 할 때마다 데이터를 읽어 온느 것이 아니라 일정한 크기의 데이터를 한번에 읽어와 버퍼에 보관 한 후, 사용자의 요청이 있을 때 버퍼에서 데이터를 읽어오는 방식으로 동작한다.


2. Scanner

공란의 줄바꿈을 모두 입력값의 경계로 인식하기 때문에 좀 더 쉽게 데이터를 입력 받을 수 있도록 해줍니다.


3. BufferedReader vs Scanner

분류 BufferedReader Scanner
Buffer Size 8192 1024
Syncronized O X
문자열 파싱 단순히 읽어 들임 문자열 파싱 가능
Exception IO 던짐 IO 숨김
