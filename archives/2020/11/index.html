Linked list (1)

Nov 01, 2020 in 이론노트, 자료구조

Linked list

  • 컴퓨터의 자료를 저장하는 여러 구조 중 하나
    일렬로 연결된 데이터를 저장 할 때 사용한다
    한 데이터의 마지막에 다음 데이터의 주소를 가지고 있음

  • 배열은 방이 정해져있기 때문에 갯수가 제한적임
    but 이거는 주소를 넣고 빼고를 통해서 조절을 할 수 있음

  • 링크를 끊게되면 자바제외 이 데이터가 남기 때문에 저장공간을 쓸데없이차지함!
    (그래서 이걸 인위적으로 지워줘야함!)

  • 이거는 주소를 하나하나 찾아 다니기 때문에 속도가 느릴수가 있음. 그래서 데이터가 정해지지않는 데이터를 넣을 때 유용함!



Linked list 단방향 양방향

단방향 : 주소를 타고 한 방향으로만 이동 가능

양방향 : 내 앞에 애가 어디있는지 알 수 있도록 앞의 주소를 가지고있음!

LinkedList 활용

앞서 작성한 단방향 코드작성은 취약점이 있다.
첫 번쨰로 시작한 header 노드가 삭제되거나 실행이 불가능하게 된다면 전체의 노드에 영향을 주기 때문이다.

이를 보완하는 방법이 존재한다. Node Class를 Linked List라는 class로 감싸서 Linked List의 header를 데이터가 아닌 시작을 알리는 용도로만 사용하도록 만든다!

그리고 그 안에 node class를 만든다!

Linked List 중복값 삭제

정렬되지 않는 LinkedList 노드중 중복값을 제거하시오.
(단 , 버퍼는 사용하지 않고!)

3 -> 2 -> 1 -> 2 -> 4

  1. 버퍼를 사용할 때

HashSet에 하나의 방에 넣고, 해당 방에 이미 노드가 존재하면 삭제한다.
( space : O(N) )
( Time : O(N) )

  1. 버퍼를 사용하지 않을 때

Pointer와 Node를 생성해서 사용한다!

( space : O(1) )
( Time : O(N^2) )
